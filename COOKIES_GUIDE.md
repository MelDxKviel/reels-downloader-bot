# Настройка cookies для YouTube и Instagram

Cookies позволяют боту действовать от имени вашего браузерного сеанса — это нужно для:

- **YouTube** — скачивание видео 18+, приватных видео, членских материалов
- **Instagram** — скачивание из закрытых аккаунтов, обход ошибок авторизации

## Требования к формату

yt-dlp принимает cookies только в **Netscape-формате** (`.txt`). Первая строка файла должна содержать `# Netscape HTTP Cookie File`.

---

## Шаг 1 — Экспорт cookies из браузера

### Chrome / Chromium / Edge / Brave

1. Установите расширение **[Get cookies.txt LOCALLY](https://chrome.google.com/webstore/detail/get-cookiestxt-locally/cclelndahbckbenkjhflpdbgdldlbecc)**
   > Используйте именно «LOCALLY» — оно не отправляет данные на сторонние серверы.
2. Откройте сайт, в котором вы авторизованы (youtube.com или instagram.com).
3. Нажмите на иконку расширения и выберите **Export** → сохраните файл.

### Firefox

1. Установите расширение **[cookies.txt](https://addons.mozilla.org/en-US/firefox/addon/cookies-txt/)**.
2. Откройте нужный сайт в авторизованном состоянии.
3. Нажмите иконку расширения → **Export** → сохраните файл.

### Проверка формата

Откройте файл в любом текстовом редакторе. Первые строки должны выглядеть примерно так:

```
# Netscape HTTP Cookie File
# https://curl.se/docs/http-cookies.html
# This file was generated by Get cookies.txt LOCALLY

.youtube.com	TRUE	/	TRUE	1735689600	VISITOR_INFO1_LIVE	xxxxx
```

## Шаг 2 — Размещение файлов

Рекомендуется хранить файлы cookies рядом с проектом в защищённом месте:

```
reels-downloader-bot/
├── cookies/
│   ├── youtube.txt
│   └── instagram.txt
├── src/
└── ...
```

Создайте директорию и скопируйте файлы:

```bash
mkdir -p cookies
cp ~/Downloads/youtube.com_cookies.txt cookies/youtube.txt
cp ~/Downloads/instagram.com_cookies.txt cookies/instagram.txt
```

> **Безопасность:** добавьте папку `cookies/` в `.gitignore`, чтобы не закоммитить учётные данные.
>
> ```bash
> echo "cookies/" >> .gitignore
> ```

---

## Шаг 3 — Настройка переменных окружения

### Запуск локально (`.env`)

Добавьте в файл `.env` в корне проекта:

```env
YT_COOKIES_FILE=cookies/youtube.txt
INSTA_COOKIES_FILE=cookies/instagram.txt
```

Задавать обе переменные необязательно — укажите только те, что нужны.

### Запуск через Docker Compose

В `docker-compose.yml` пробросьте папку cookies как том и укажите переменные:

```yaml
services:
  bot:
    environment:
      YT_COOKIES_FILE: /app/cookies/youtube.txt
      INSTA_COOKIES_FILE: /app/cookies/instagram.txt
    volumes:
      - ./cookies:/app/cookies:ro
```

Флаг `:ro` монтирует папку в режиме «только чтение» — дополнительная защита.

---

## Как это работает

| Переменная | Платформа | Когда нужна |
|---|---|---|
| `YT_COOKIES_FILE` | YouTube | Видео 18+, приватные видео, материалы для участников |
| `INSTA_COOKIES_FILE` | Instagram | Закрытые аккаунты, обход ошибок `login_required` |

- Cookies передаются в yt-dlp только для соответствующей платформы.
- Перед использованием бот проверяет, что файл существует и имеет Netscape-формат.
- Если файл невалиден — выводится предупреждение в лог, скачивание продолжается без cookies.
- Для Instagram дополнительно работает fallback через зеркало kkinstagram.com, если скачивание с cookies не удалось.

## Устранение проблем

**Ошибка `does not look like a Netscape format cookies file`**
— файл экспортирован в неверном формате. Убедитесь, что первая строка начинается с `# Netscape HTTP Cookie File`.

**Cookies не применяются, в логах нет ошибок**
— проверьте, что переменная окружения задана и путь указан правильно. Относительные пути отсчитываются от директории запуска бота.

**Ошибка `Sign in` или `login_required` несмотря на cookies**
— сеанс браузера истёк или cookies устарели. Войдите в аккаунт заново и повторите экспорт.

**YouTube: «Sign in to confirm you're not a bot»**
— YouTube усилил проверки. Попробуйте экспортировать cookies из браузера без VPN, или используйте опцию `--cookies-from-browser` в yt-dlp вручную для проверки.
